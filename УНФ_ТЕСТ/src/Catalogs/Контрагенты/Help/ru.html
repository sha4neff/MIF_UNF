Справочник "Контрагенты"

Справочник предназначен для хранения списка контрагентов. Контрагент - это общее понятие, куда включены поставщики и покупатели, организации и частные лица.

 

Контрагенты могут быть сгруппированы по группам классификатора справочника, например, "Поставщики", "Покупатели" и "Реализаторы". Справочник контрагентов имеет неограниченное количество уровней вложенности.

 

Редактирование информации о контрагентах производится в отдельном диалоговом окне, открываемом из формы списка справочника контрагентов.

При этом указывается информация, характеризующая контрагента:

  • Краткое наименование. Используется для выбора и быстрого поиска контрагента в списках.
  • Полное наименование контрагента. Используется для отражения в печатных формах документов.
  • Группа. Ссылка на группу справочника, в которую включен элемент. При ручном изменении реквизита "Группа" происходит перемещение текущего элемента из старой группы в новую.
  • Код. Код элемента контрагента.
  • Вид контрагента. Указывается один из вариантов: юрлицо или физлицо.
  • Физ. лицо. Указывается только для физического лица. Выбирается из справочника "Физические лица".
  • ИНН - идентификационный номер налогоплательщика представляет собой цифровой код, упорядочивающий сбор налогов с налогоплательщиков в Российской Федерации. ИНН присваивается как юридическим, так и физическим лицам.  Используется для отражения в печатных формах.
  • КПП - код причины постановки на учет используется только для юридических лиц и является дополнением к ИНН в связи с постановкой на учет в разных налоговых органах. Используется для отражения в печатных формах документов.
  • ОКПО - код по общероссийскому классификатору предприятий и организаций присваивается в момент регистрации юридического лица и показывает вид деятельности. Используется для отражения в печатных формах документов.
  • Рег. номер - для иностранной организации можно указать регистрационный номер, присвоенный в стране регистрации
  • Значения по умолчанию. Группа реквизитов:
    • Банковский счет по умолчанию, для подстановки в документы. Выбирается из справочника "Банковские счета".
    • Договор по умолчанию, для подстановки в документы. Выбирается из справочника "Договоры контрагентов".
  • Учет взаиморасчетов. Группа реквизитов:
    • Аналитика взаиморасчетов с контрагентом. Выбирается в колонке "Долги": "по договорам", "по накладным" или "по заказам". Установка флага позволяет выводить в отчёте долги с аналитикой по указанному измерению. Также установка флага приводит к тому, что в документах требыутся выбирать договор, документ или заказ.
    • Флажки в колонке "Оплата". Установка флажков позволяет видеть в списке заказов, счетов на оплату или накладных информацию об оплате в виде кружков.
  • Счета учета. Группа реквизитов панели навигации формы:
    • Расчеты с покупателем. Выбирается из Плана счетов "Управленческий". Требуется обязательное заполнение.
    • Авансы покупателя.  Выбирается из Плана счетов "Управленческий". Требуется обязательное заполнение.
    • Расчеты с поставщиком.  Выбирается из Плана счетов "Управленческий". Требуется обязательное заполнение.
    • Авансы поставщику.  Выбирается из Плана счетов "Управленческий". Требуется обязательное заполнение.
  • Группа доступа. Группа доступа контактов и контрагентов, определяющая ограничение доступа пользователей к элементам справочника и документам по контрагенту.

На закладке "Адреса, телефоны" указывается контактная информация контрагента: E-mail, другое, почтовый адрес, телефон, факс, фактический адрес и юридический адрес.

На закладке "Дополнительная информация" указывается любая дополнительная информация о контрагенте.

При нажатии кнопки "Дополнительные реквизиты" открывается диалоговое окно редактирования дополнительных реквизитов, определяемых пользователем.

 

Заполнение реквизитов контрагента по ИНН

По кнопке "Заполнить по ИНН" происходит заполнение реквизитов контрагента. Заполнение производится на основе данных ЕГРЮЛ и ЕГРИП. При наличии соответствующей информации, будут заполнены следующие реквизиты:

для юридического лица:

- КПП;
- краткое и полное наименование;
- руководитель (будет создано контактное лицо);
- юридический адрес;
- фактический адрес (если не заполнен);
- телефон;

для физического лица (предпринимателя):
- краткое, полное наименование;
- физическое лицо (будет создано).

Дополнительно можно использовать проверку контрагентов по данным ФНС. Настройка проверки осуществляется в разделе Настройки - Настройки проверки контрагентов. Пользователям "1С:Предприятия через интернет" сервис подключается автоматически.

 

Расширенный поиск контрагентов

Для форм списка и выбора контрагентов доступен функционал расширенного поиска контрагентов, если в сервисных настройках включено использование полнотекстового поиска.

Расширенный поиск позволяет найти список контрагентов, для которых либо в самой форме, либо в связанной с контрагентом информации найдена заданная строка.

Связанной с контрагентом информацией считаются справочники "Контакты", "Физические лица", "Договоры контрагентов" и "Банковские счета".

Важно. Следует учитывать, что некоторые символы, введенные в строку расширенного поиска, будут трактоваться как служебные символы языка запросов полнотекстового поиска в данных. Информацию о списке таких служебных символов можно получить в справке (раздел "1С:Предприятие" - "Работа пользователя" - "Язык запросов полнотекстового поиска в данных").

После выполнения поиска в списке контрагентов остаются только те, которые удовлетворяют условию поиска.

При позиционировании на контрагенте внизу на форме отображается информация о том, в каком объекте было найдено совпадение (важно: отображается первый найденный объект) и связь данного объекта с контрагентом. При нажатии на гиперссылку в информационной строке откроется форма объекта, в котором было найдено совпадение.

В случае если не один контрагент найден не будет, поле ввода строки поиска будет подсвечено красным.

Для того чтобы отменить отображение списка по результатам расширенного поиска, необходимо очистить строку поиска.
